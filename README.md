# 🍺 Таверна Дуэль

Онлайн авто-баттлер на двоих в духе **Hearthstone Battlegrounds**. Создаёшь комнату,
кидаешь корешу ссылку — и рубитесь. Чистый Node, **без npm-зависимостей**, деплой одним
файлом на Render.

## Как играть

1. Открой сайт → **Создать игру**.
2. Скинь корешу ссылку (кнопка «Скопировать ссылку») или 4-значный код.
3. Каждый ход в **таверне**: получаешь золото, покупаешь миньонов из лавки (3🪙),
   продаёшь (+1🪙), реролл лавки (1🪙), апаешь тир таверны — и расставляешь борд (до 7).
4. Жмёшь **«В БОЙ»** — борды автоматически рубятся. Проигравший теряет HP.
5. Довёл HP соперника до 0 — победа. 🏆

### Ключевики миньонов
- 🛑 **Провокация** — должен быть атакован первым.
- ✨ **Щит** — поглощает первый удар.
- 💨 **Вихрь** — атакует дважды.
- ☠️/🐣 **Предсмертный хрип** — при гибели призывает токенов.

## Запуск локально

```bash
node server.mjs        # → http://localhost:3000
npm test               # юнит-тесты боя и игрового автомата
```

Порт берётся из `PORT` (Render задаёт сам), иначе 3000.

## Архитектура

| Файл | Ответственность |
|------|------------------|
| `server.mjs` | HTTP + SSE, маршрутизация, таймеры фаз |
| `lib/cards.mjs` | пул миньонов, лавка, инстансы |
| `lib/combat.mjs` | чистый резолвер боя (детерминирован по сиду) |
| `lib/game.mjs` | состояние комнаты, действия, переходы фаз |
| `public/` | клиент: SSE, рендер вербовки, анимация боя |

Real-time без зависимостей: **SSE** (сервер→клиент) + **POST /api/action** (клиент→сервер).
Состояние комнат — в памяти процесса.

## Деплой на Render

Репозиторий содержит `render.yaml` (Blueprint). Подключаешь репо в Render как Blueprint —
сервис `tavern-duel` поднимается сам (`node server.mjs`, без билд-шага).

> ⚠️ Free-план Render засыпает после простоя — первый заход после паузы стартует ~30 сек.
